<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repo to CI runner | CI executes repo scripts and third-party actions; a poisoned script, action tag or dependency runs here |
| CI/Netlify to public | The deploy publishes only what passed the gates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | A bad commit deploys because the gates run only in tests | mitigate | Netlify build command is npm run build, so prebuild gates run at deploy time; CI also runs them on every push |
| T-08-02 | Tampering | Supply-chain code runs in CI | mitigate | npm ci against the committed lockfile; npm audit --omit=dev --audit-level=high fails on a known-high CVE in the PRODUCTION tree (the full dev tree carries routine puppeteer-chain advisories with zero runtime relevance since the output is static HTML, so dev is scoped out to avoid a day-one false failure); no npx of unverified packages; pinned Node 24 |
| T-08-05 | Tampering | A mutable action tag (@v4) is repointed to malicious code (tj-actions / Shai-Hulud 2025) | mitigate | actions/checkout and actions/setup-node pinned to 40-char commit SHAs RESOLVED from the real tags via gh api / git ls-remote (never a fabricated hex string, which would pass a regex but error at run time; offline, resolution becomes a human checkpoint); top-level permissions contents:read; Dependabot keeps the pins current |
| T-08-03 | Information disclosure | A malicious PR deploy-preview build reads a build-time secret | accept (low) | Phase 1 has no build-time secrets (the Cloudflare analytics token is public, not a secret; OFF keys are Phase 2), so a deploy-preview build can read nothing. FUTURE TRIGGER: once any build-time env var/secret exists (Phase 2 ingestion), revisit — a malicious PR's deploy-preview could read it; restrict preview builds then |
| T-08-04 | Tampering | Weak CSP / missing HSTS allows injected scripts or transport downgrade | mitigate | Tightened CSP + HSTS from plan 01-01 verified on the LIVE deploy response in Task 3 |
</threat_model>
