<objective>
Establish the DATA CONTRACTS for the whole phase: the inline `verification` record on every `SourcedValue`, the `derivedFrom` lineage tag on source records, the distinct `lead` schema for OFF imports, and the negative fixtures every downstream gate proves itself against. This is an interface-first Wave 0: downstream plans (02-02 logic, 02-03 gate, 02-04 ingestion, 02-05 checker, 02-06 audit) implement AGAINST these shapes rather than exploring for them.

Purpose: A "pass is a verification event, not a source" (D-02) is a data-shape decision, not a library one. Locking the shapes first lets the three seams build in parallel.
Output: Extended `sourced-value` and `source` schemas, a new `lead` schema, seven fixtures, one schema test, and the three standalone npm script entries the later network scripts register under.
</objective>
