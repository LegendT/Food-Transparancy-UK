Signatures to export from lib/verification.mjs (full bodies in 02-RESEARCH.md Pattern 2 and Pattern 5):
- deriveVerificationState(fact, sourcesById, existenceBySourceId, today) -> { state, reasons }
    where existenceBySourceId is Map<sourceId, { verdict, resolvedVia, checkedAt }> loaded from the cache;
    verdict "RESOLVES" is the only value that satisfies the existence precondition (D-07).
- meetsCorroborable(confirmsPasses, fact, sourcesById) -> boolean  (>=2 confirms, >=2 distinct lineages, >=1 primary by registry sourceType)
- meetsAuthoritative(confirmsPasses) -> boolean  (>=1 authority pass + >=1 re-read with distinct reviewerKind; D-06 - distinct-lineage does NOT apply)
- checkDistinctLineage(facts, sources) -> { errors }   (fires only for claimType === "corroborable"; co-derived sources sharing a derivedFrom root count as ONE lineage)
- lineageSimilarityWarnings(facts, sources) -> { warnings }  (non-blocking: same publisher OR same eTLD+1 but no derivedFrom declared)
- checkMeasureMismatch(passes) -> boolean   (deep-equal structured measure across confirms passes; any inequality => true)
- classifyStaleness(fact) -> "regulatory" | "current" | "historical"  (D-16 derivation)
- isPastStaleness(fact, lastVerified, today) -> boolean  (regulatory 12mo / current 24mo / historical never)
- lastVerified(confirmsPasses) -> ISO date string (max checkedOn)

Fact shape (from 02-01 schema): fact.verification.passes[] { reviewerKind, sourcesChecked[], measure{basis,state,...}, checkedValue?, verdict, checkedOn }; fact.verification.adjudication?.outcome; fact.verification.contested?.positions[]; fact.verification.markedWrong?; fact.verification.stalenessClass?; fact.claimType; fact.claimDomain; fact.sources[].

D-15 precedence (highest first): markedWrong -> withheld-wrong; adjudication.outcome==="contested" -> published-contested; any cited source not RESOLVES -> withheld-unverified; any disputes verdict OR measure mismatch -> withheld-open-disagreement; insufficient passes for claim type -> withheld-unverified; past staleness -> published-stale; else published-confirmed.

D-16 staleness derivation: stalenessClass override ?? (claimDomain === "regulatory" ? "regulatory" : isTimelineEntity(fact) ? "historical" : "current"). A product-embedded past-formulation fact that is not a timeline entity carries an explicit stalenessClass override.
