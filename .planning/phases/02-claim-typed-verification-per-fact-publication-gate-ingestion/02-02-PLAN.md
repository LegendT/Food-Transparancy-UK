Signatures to export from lib/verification.mjs (full bodies in 02-RESEARCH.md Pattern 2 and Pattern 5):
- CITATION_TTL_DAYS = 180  (a fixed ceiling for v1; a cached RESOLVES verdict whose checkedAt is older than this many days before `today` is treated as UNCHECKED. SEAM pin: the audit agent SURFACES this same 180-day constant; the gate/derivation ENFORCES it. Ponytail comment: v1 uses one flat ceiling; the upgrade path is a per-staleness-class TTL keyed off classifyStaleness.)
- deriveVerificationState(fact, sourcesById, existenceBySourceId, today, entityType) -> { state, reasons }
    where existenceBySourceId is Map<sourceId, { verdict, resolvedVia, checkedAt, statusCode, snapshotUrl }> loaded from the shared cache (SEAM pin, snapshotUrl nullable);
    verdict "RESOLVES" with a fresh checkedAt (within CITATION_TTL_DAYS) is the only value that satisfies the existence precondition (D-07/R-07);
    entityType is a hint (e.g. "timeline-event" | "product" | ...) or the instance path, threaded to classifyStaleness (R-17).
- meetsCorroborable(confirmsPasses, fact, sourcesById) -> boolean  (>=2 confirms, >=2 distinct lineages, >=1 primary by registry sourceType)
- meetsAuthoritative(confirmsPasses) -> boolean  (>=1 authority pass + >=1 re-read with distinct reviewerKind; D-06 - distinct-lineage does NOT apply)
- checkDistinctLineage(facts, sources) -> { errors }   (fires only for claimType === "corroborable"; co-derived sources sharing a derivedFrom root count as ONE lineage; a cycle canonicalises to the lexicographically minimal id in the cycle, R-16)
- lineageSimilarityWarnings(facts, sources) -> { warnings }  (non-blocking: same publisher OR same eTLD+1 but no derivedFrom declared)
- checkMeasureMismatch(passes) -> boolean   (deep-equal structured measure across confirms passes; any inequality => true)
- checkValueDivergence(passes) -> boolean   (among confirms passes that share a measure, any inequality in the (checkedValue, unit) tuple => true; exact equality for v1, tolerance is an upgrade path, R-04)
- classifyStaleness(fact, entityType) -> "regulatory" | "current" | "historical"  (D-16 derivation, entityType hint per R-17)
- isPastStaleness(fact, lastVerified, today, entityType) -> boolean  (regulatory 12mo / current 24mo / historical never; a future lastVerified is NOT fresh, R-25a)
- lastVerified(confirmsPasses) -> ISO date string (max checkedOn)

Fact shape (from 02-01 schema): fact.verification.passes[] { reviewerKind, sourcesChecked[], measure{basis,state,unit?,...}, checkedValue?, verdict, checkedOn }; fact.verification.adjudication?.{outcome, note, date, correctedValue?}; fact.verification.contested?.positions[]{value,sources,note}; fact.verification.markedWrong?; fact.verification.stalenessClass?; fact.claimType; fact.claimDomain; fact.sources[].

D-15 precedence (highest first), as refined by R-02/R-03/R-04/R-07/R-14/R-15:
  1. markedWrong -> withheld-wrong.
  2. adjudication.outcome === "contested" -> published-contested, BUT only after the contested.positions[].sources all satisfy the RESOLVES precondition; a non-RESOLVES position degrades (withhold rather than publish a dead citation, R-14).
  3. any source cited by the CONFIRMS passes that would establish sufficiency (NOT all passes, so a stray inaccessible/disputes pass does not falsely withhold, R-15) whose cached verdict is not a fresh RESOLVES (non-RESOLVES, absent, or a RESOLVES older than CITATION_TTL_DAYS) -> withheld-unverified (D-07/R-07/R-08).
  4. any pass verdict "disputes" OR checkMeasureMismatch OR checkValueDivergence over confirms passes -> withheld-open-disagreement, UNLESS a human adjudication.outcome "confirmed" or "corrected" dated on/after the latest disputing/mismatching pass clears it (R-03): "confirmed" resumes normal sufficiency; "corrected" substitutes adjudication.correctedValue as the fact value and resumes sufficiency.
  5. insufficient passes for claim type -> withheld-in-review when the fact carries >=1 recorded pass but is still below its claim-type bar, else withheld-unverified when it has zero passes (the VRFY-03 workflow split; both remain unpublished).
  6. past staleness -> published-stale.
  7. else published-confirmed.

D-16 staleness derivation: stalenessClass override ?? (claimDomain === "regulatory" ? "regulatory" : isTimelineEntity(entityType, fact) ? "historical" : "current"). A product-embedded past-formulation fact that is not a timeline entity carries an explicit stalenessClass override.
