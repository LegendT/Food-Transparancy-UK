<objective>
Build the Open Food Facts ingestion path: `scripts/ingest-off.mjs` reads a barcode worklist, fetches the OFF v2 per-barcode API with field selection, and writes provenance-tagged draft leads into `ingestion/leads/` under the distinct `lead` schema. Leads are raw imported data, NOT facts (D-19): they live outside `src/_data`, outside the Eleventy cascade, outside the validation corpus, and never use the key `sources`. Promotion (turning a lead into a `SourcedValue` fact) is an explicit human step out of scope here. The OFF revision (`rev`) is recorded so a revision-diff can be captured as an `off-revision-diff` lead; Phase 2 records the lead and the never-publish guard only, not an automated differ.

Purpose: Seed the draft store with real current-formulation data while structurally guaranteeing OFF (tertiary, crowd-sourced) can never reach a published page without passing the full verification gate.
Output: One network script, a barcode worklist, the isolated lead directory, and two offline tests.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Tampering | Untrusted OFF data promoted as authority | mitigate | Leads are isolated (D-19), never rendered; promotion is a human step; a minted fact still faces the full gate and OFF (tertiary) can never satisfy the corroborable standard alone |
| T-02-04-02 | Information disclosure | SSRF via a malformed barcode/host in the worklist | mitigate | Fetch only https://world.openfoodfacts.org; refuse any other host; no user-controlled URL construction beyond the barcode path segment (encodeURIComponent the barcode) |
| T-02-04-03 | Denial of service | Oversized/slow OFF response exhausting the build | mitigate | AbortSignal.timeout(8000) per request; field-selection query limits payload; ~1-2 req/s cap |
| T-02-04-04 | Tampering | Malicious OFF text interpolated into code | mitigate | OFF fields are stored as data only, never eval'd or interpolated into a shell/template; Ajv additionalProperties:false on the lead shape |
| T-02-04-SC | Tampering | npm/pip/cargo installs | accept | Zero new dependencies (native fetch); no install checkpoint applies |
</threat_model>
