<objective>
Build the Open Food Facts ingestion path: `scripts/ingest-off.mjs` reads a barcode worklist, fetches the OFF v2 per-barcode API with field selection, and writes provenance-tagged draft leads into `ingestion/leads/` under the distinct `lead` schema. Leads are raw imported data, NOT facts (D-19): they live outside `src/_data`, outside the Eleventy cascade, outside the validation corpus, and never use the key `sources`. Promotion (turning a lead into a `SourcedValue` fact) is an explicit human step out of scope here, but the lead records the `off` source registry id so a promotion is obligated to cite it and keep the ODbL attribution/share-alike link (R-09). The OFF revision (`rev`) is recorded so a revision-diff can be captured as an `off-revision-diff` lead; Phase 2 records the lead and the never-publish guard only, not an automated differ.

Purpose: Seed the draft store with real current-formulation data while structurally guaranteeing OFF (tertiary, crowd-sourced) can never reach a published page without passing the full verification gate and preserving its ODbL obligations.
Output: One network script, a barcode worklist, the isolated lead directory, and two offline tests.
</objective>

  not found: { "code": "<bc>", "status": 0, "status_verbose": "product not found" }  (HTTP 404)
Send a descriptive User-Agent: "FoodTransparencyUK/0.1 (mailto:legendarytone@gmail.com)".

Measure basis (R-24, RESEARCH A2 - read the ACTUAL suffix, never assume):
  - a nutriment carrying *_100g -> measure.basis "per-100g" (state "as-sold")
  - only *_serving present -> measure.basis "per-serving", reading product.nutrition_data_per ("serving") to confirm the basis
  - neither present -> record the field with NO measure (a promotion then supplies one)

Body-size cap (R-23): AbortSignal.timeout bounds TIME, not bytes, and res.json() buffers the whole auto-decompressed body (gzip-bomb surface). Stream res.body and abort past a byte ceiling (512 KB) before parsing JSON; keep the time abort as well.

lead record shape (from schemas/lead.schema.json, created in 02-01):
  { leadId, leadType: "off-import"|"off-revision-diff", barcode, capturedAt, offRevision,
    provenance: { dataset:"open-food-facts", sourceRegistryId:"off", licence:"ODbL-1.0", url, retrievedAt },
    fields: [ { path, offField, value, measure?:{basis,state}, offRevision } ],
    promotion: { status:"pending", note:"", promotedTo:null, by:null, date:null },
    revisionDiff: null }
provenance.sourceRegistryId records the off registry id (R-09): a promotion MUST cite it so the ODbL attribution/share-alike link is not dropped; the 02-06 audit surfaces any OFF-derived fact that has lost the link.

The lead sample fixture test/fixtures/valid/off-response.sample.json (created in 02-01) is the offline input for the ingestion unit test; the per-serving branch is exercised with an inline product object in the test (no new shared fixture).
