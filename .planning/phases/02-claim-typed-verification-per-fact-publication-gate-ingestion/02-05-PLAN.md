<objective>
Build the four-verdict citation-existence checker (VRFY-07). Split the PURE decision from the IO: `lib/citation-status.mjs` holds `classifyStatus(statusCode, errorClass)` (unit-testable, no network), and `scripts/check-citations.mjs` performs the fetch escalation (HEAD -> GET Range on refusal -> Wayback CDX fallback -> Crossref/Handle for DOIs) and writes a diffable verdict cache. The four-verdict model (RESOLVES / DOES_NOT_RESOLVE / ACCESS_BLOCKED / INDETERMINATE) exists because a boolean produced the exact false-block failure SPIKE-01 exposed: our real citations are bot-hostile (retailers 403, Wayback rate-limits from CI IPs), and a live-but-refused host must never be scored dead.

Purpose: "Every recorded citation resolves before any pass counts." The gate reads the cache; a non-RESOLVES verdict withholds the fact. The existence check reaches the network but only ever records a resolves-verdict, never a value and never a pass (D-11).
Output: One pure module, one standalone network script, one unit test. The gate (02-03) and audit (02-06) consume the cache format documented in 02-01.
</objective>

classifyStatus(statusCode, errorClass) -> verdict  (D-07 table, verified):
  errorClass "nxdomain"|"refused" -> DOES_NOT_RESOLVE
  errorClass "timeout"|"network"  -> INDETERMINATE
  status 200-399 (incl 206, 304)  -> RESOLVES
  status 404|410                  -> DOES_NOT_RESOLVE
  status 401|403|407|429|451|999  -> ACCESS_BLOCKED
  status >= 500                   -> INDETERMINATE
  else                            -> INDETERMINATE

Escalation order (D-08), each step verified reachable server-side:
  1. LIVE: HEAD, realistic browser User-Agent, redirect:"follow" cap ~10, AbortSignal.timeout(8000)
  2. on 403/405/429 retry ONCE as GET with header Range: bytes=0-0; honour Retry-After once, bounded
  3. ACCESS_BLOCKED/INDETERMINATE/DOES_NOT_RESOLVE -> Wayback CDX (PREFER CDX; availability API 429s shared IPs):
     https://web.archive.org/cdx/search/cdx?url={url}&output=json&limit=1&filter=statuscode:200 [&closest={ts}]
     a 200 snapshot row -> verdict RESOLVES, resolvedVia "wayback:{timestamp}", store the snapshot URL
  4. DOI (starts with "10."): Crossref HEAD https://api.crossref.org/works/{doi} {200|404}
     AND Handle https://doi.org/api/handles/{doi} {responseCode 1|100}
     -> RESOLVES if Crossref 200 OR Handle 1; DOES_NOT_RESOLVE only if Crossref 404 AND Handle 100; else INDETERMINATE
  Crossref User-Agent: "FoodTransparencyUK/0.1 (mailto:legendarytone@gmail.com)".

Known bot-hostile hosts (bias refusals to ACCESS_BLOCKED + Wayback fallback): tesco.com, ocado.com, asda.com, sainsburys.co.uk, web.archive.org.

Cache format (documented in test/fixtures/valid/citation-verdicts.sample.json from 02-01):
  { "<sourceId>": { "verdict", "resolvedVia": "live"|"wayback:<ts>"|"crossref"|"handle", "checkedAt", "statusCode" } }
  written to .cache/citation-verdicts.json (COMMIT it; diffable; the offline gate reads it).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| editor-supplied source URL -> outbound request | The checker turns trusted-today source URLs into server-side fetches |
| external hosts -> build tooling | Untrusted HTTP responses (status, headers, body) drive verdicts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Information disclosure / EoP | SSRF via a crafted source URL probing the internal network from CI | mitigate | https-only (plus the archive http host); refuse private/loopback/link-local hosts by parsing with URL and checking the resolved host before fetch (Pitfall 6) |
| T-02-05-02 | Denial of service | Oversized/slow response, redirect loop, decompression bomb | mitigate | AbortSignal.timeout(8000); redirect cap ~10; HEAD/GET Range bytes=0-0 (never download full bodies); ~1-2 req/s cap |
| T-02-05-03 | Spoofing | A live-but-refused host (403/429) mis-scored as a dead link | mitigate | Four-verdict classifier maps refusals to ACCESS_BLOCKED, never DOES_NOT_RESOLVE (D-07); Wayback fallback; residue routed to a human |
| T-02-05-04 | Tampering | Cache poisoning of .cache/citation-verdicts.json | mitigate | Cache is committed and diff-reviewed like any data file; a bad verdict is a visible git diff; the gate re-derives transparently |
| T-02-05-05 | Elevation of Privilege | An automated existence-RESOLVES treated as a verification pass | mitigate | The script records only resolves-verdicts, never a value or a pass; passes stay editor-authored (D-11) |
| T-02-05-SC | Tampering | npm/pip/cargo installs | accept | Zero new dependencies (native fetch); no install checkpoint applies |
</threat_model>
