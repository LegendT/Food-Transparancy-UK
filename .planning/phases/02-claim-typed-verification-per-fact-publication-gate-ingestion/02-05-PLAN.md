<objective>
Build the four-verdict citation-existence checker (VRFY-07). Split the PURE decisions from the IO: `lib/citation-status.mjs` holds the pure helpers (`classifyStatus`, the SSRF host-safety guard, the DOI-to-API path normaliser, the closest-snapshot picker, the soft-404 heuristic - all unit-testable with no network), and `scripts/check-citations.mjs` performs the fetch escalation (HEAD -> GET Range on refusal -> Wayback CDX fallback -> Crossref/Handle for DOIs) and writes a diffable verdict cache. The four-verdict model (RESOLVES / DOES_NOT_RESOLVE / ACCESS_BLOCKED / INDETERMINATE) exists because a boolean produced the exact false-block failure SPIKE-01 exposed: our real citations are bot-hostile (retailers 403, Wayback rate-limits from CI IPs), and a live-but-refused host must never be scored dead.

The checker turns editor-supplied source URLs into outbound requests, so it is an SSRF surface hardened here: it follows redirects manually and re-runs the full host-safety policy against every hop (R-01), the policy canonicalises the host across all IP encodings rather than string-matching (R-11), and every fetch is body-size-capped (R-23). DOIs are sent with the forward slash intact (R-10), and a Wayback snapshot that mitigates a blocked origin is persisted durably so the citation does not flip back to withheld on the next archive.org 429 (R-19).

Purpose: "Every recorded citation resolves before any pass counts." The gate reads the cache; a non-RESOLVES verdict withholds the fact. The existence check reaches the network but only ever records a resolves-verdict, never a value and never a pass (D-11).
Output: One pure helper module, one standalone network script, one unit test. The gate (02-03) and audit (02-06) consume the cache format documented in 02-01.
</objective>

classifyStatus(statusCode, errorClass) -> verdict  (D-07 table, verified; errorClass extended per R-21):
  errorClass "nxdomain"|"refused"                      -> DOES_NOT_RESOLVE   (only these two are affirmative non-existence)
  errorClass "dns-transient"|"tls"|"timeout"|"network" -> INDETERMINATE      (route to human, never dead)
  any other/unknown errorClass                         -> INDETERMINATE
  status 200-399 (incl 206, 304)                       -> RESOLVES
  status 404|410                                       -> DOES_NOT_RESOLVE
  status 401|403|407|429|451|999                       -> ACCESS_BLOCKED
  status >= 500                                        -> INDETERMINATE
  else                                                 -> INDETERMINATE

Thrown-error -> errorClass mapping the SCRIPT applies before calling classifyStatus (R-21):
  ENOTFOUND -> "nxdomain"; ECONNREFUSED -> "refused";
  EAI_AGAIN -> "dns-transient";
  CERT_HAS_EXPIRED / DEPTH_ZERO_SELF_SIGNED_CERT / UNABLE_TO_VERIFY_LEAF_SIGNATURE / any TLS/cert code -> "tls";
  ETIMEDOUT / AbortError (AbortSignal.timeout) -> "timeout";
  anything else -> "network".
  Only ENOTFOUND and ECONNREFUSED may become DOES_NOT_RESOLVE; TLS/transient-DNS/timeout/unknown are INDETERMINATE.

SSRF host-safety policy (R-01 + R-11) - a pure helper, applied to the INITIAL url AND to every redirect hop:
  assertPublicHttpsUrl(rawUrl): new URL(rawUrl); scheme MUST be https (the only http exception is the archive host web.archive.org used by the CDX probe).
  isBlockedHost(hostname): canonicalise the host and REFUSE if it is any of:
    - a private/loopback/link-local/ULA IP, parsed to canonical bytes across ALL encodings:
        IPv4 dotted (127.0.0.1, 10/8, 172.16/12, 192.168/16, 169.254/16, 0.0.0.0),
        IPv4 decimal/octal/hex (2130706433, 0x7f000001, 0177.0.0.1),
        IPv6 loopback ::1 and [::1], ::, [::], ULA fc00::/7, link-local fe80::/10,
        IPv4-mapped IPv6 ::ffff:169.254.169.254 (and bracketed forms);
    - the cloud metadata hostnames metadata.google.internal and 169.254.169.254 (GCP/AWS/Azure metadata);
    - a non-hostname / malformed host.
  DNS-rebinding (a public hostname that resolves to a private IP at fetch time) is a CONSCIOUS accepted risk for a build-time tool over a curated source list - documented, not mitigated here.

Escalation order (D-08), each step verified reachable server-side:
  1. LIVE: HEAD, realistic browser User-Agent, redirect:"manual" (NOT "follow"), AbortSignal.timeout(8000).
     On a 3xx, read the Location header, resolve it against the current url, re-run assertPublicHttpsUrl + isBlockedHost on the new host, and only then fetch the next hop. Cap the redirect count yourself at 5; exceeding the cap -> INDETERMINATE. A hop that fails the host policy -> refuse (verdict NOT RESOLVES). (R-01)
  2. on 403/405/429 retry ONCE as GET with header Range: bytes=0-0; honour Retry-After once, bounded.
  3. ACCESS_BLOCKED/INDETERMINATE/DOES_NOT_RESOLVE -> Wayback CDX (PREFER CDX; availability API 429s shared IPs):
     https://web.archive.org/cdx/search/cdx?url={url}&output=json&filter=statuscode:200&sort=closest&closest={ts}&limit=1
     NOTE (R-20): a bare &limit=1&filter=statuscode:200 returns the FIRST/oldest 200 snapshot, not the nearest. Use &sort=closest&closest={ts} (verify filter+sort compatibility) OR fetch a small unfiltered window and pick the closest statuscode:200 row client-side with pickClosestSnapshot. A 200 snapshot row -> verdict RESOLVES, resolvedVia "wayback:{timestamp}", and STORE snapshotUrl in the cache entry (R-19).
  4. DOI (id or url starting "10."): Crossref HEAD https://api.crossref.org/works/{doiPath}
     AND Handle https://doi.org/api/handles/{doiPath} where doiPath = normaliseDoiForApi(doi) keeps the forward slash intact (R-10)
     -> RESOLVES if Crossref 200 OR Handle responseCode 1; DOES_NOT_RESOLVE only if Crossref 404 AND Handle 100; else INDETERMINATE
  Crossref User-Agent: "FoodTransparencyUK/0.1 (mailto:legendarytone@gmail.com)".

Body-size cap (R-23): AbortSignal.timeout bounds TIME, not bytes; res.json()/res.text() buffer the whole auto-decompressed body (gzip-bomb surface on the Crossref/Handle/CDX JSON paths and the GET-Range path). Stream res.body and abort past a byte ceiling (512 KB) before parsing. Do NOT claim a Range header prevents this - a server may ignore Range and stream a full body; the ceiling is the real bound.

normaliseDoiForApi(doi) -> string: encodeURIComponent(doi).replace(/%2F/gi, "/") so "10.1038/nature12373" stays "10.1038/nature12373", never "10.1038%2Fnature12373" (which 404s, R-10).

Soft-404 heuristic (R-22): classifyStatus is status-only, so a 200 "page no longer available" would score RESOLVES. On the GET path apply isSoftNotFound(status, bodyText): a 200 whose (byte-capped) body is very small OR contains a common not-found marker in the title/body -> treat as INDETERMINATE (route to human), never RESOLVES.

Known bot-hostile hosts (bias refusals to ACCESS_BLOCKED + Wayback fallback): tesco.com, ocado.com, asda.com, sainsburys.co.uk, web.archive.org.

Cache format (SEAM - documented in test/fixtures/valid/citation-verdicts.sample.json from 02-01; the gate 02-03 and audit 02-06 read the SAME shape):
  { "<sourceId>": { "verdict", "resolvedVia": "live"|"wayback:<ts>"|"crossref"|"handle", "checkedAt": ISO-8601 date-time, "statusCode": number|null, "snapshotUrl": string|null } }
  snapshotUrl is the Wayback capture URL when one mitigated an ACCESS_BLOCKED origin, else null (R-19).
  checkedAt is an ISO-8601 date-time; the gate treats an entry older than the shared CITATION_TTL_DAYS (180, defined in lib/verification.mjs) as UNCHECKED, and the audit surfaces it as due-for-recheck.
  Written to .cache/citation-verdicts.json (COMMIT it; diffable; the offline gate reads it).
