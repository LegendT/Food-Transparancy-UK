<objective>
Build the re-verification audit command (`scripts/audit-verification.mjs`): a read-only report generator that walks the corpus, derives each fact's publication state and staleness class via `lib/verification.mjs`, reads the committed existence-verdict cache, and emits a dated, worst-first audit record modelled exactly on DEBT's `docs/DATA-AUDIT.md`. The document is the human triage surface (a counts-by-status table that reports the VRFY-03 workflow states distinctly - including withheld-in-review separately from withheld-unverified - then discrepancy sections ordered wrong -> disagreement -> stale-oldest-first, each carrying the fact's recorded structured measure and pass verdicts, plus a separate reviewer-disagreements section). The audit NEVER writes a verdict or a value: the human adjudicates by editing the fact inline (D-04/D-17). A fact past its staleness threshold is flagged with its last-verified date, which is the data the Phase 3a reader-facing "last verified {date} - review due" indicator (VRFY-12) will render.

Purpose: "Verified and re-verified until we are sure." Staleness becomes a generated queue, not a good intention.
Output: One standalone read-only script and its test. This plan owns scripts/audit-verification.mjs exclusively.
</objective>

Audit document shape (from DEBT DATA-AUDIT.md, verified template):
  # Food Transparency Data Verification Audit
  **Date:** <ISO or long-form date>
  ## Counts by status         (one row per derived state)
  | Status | Count |
  ## Discrepancies to approve  (ordered worst-first)
  ### Wrong (n)  / ### Open disagreement (n) / ### Stale (n)
  per entry: fact path, current value, the recorded structured measure, the recorded pass verdicts, last-verified date
  ## Reviewer disagreements - flag for extra human scrutiny   (facts with a disputes verdict or a measure mismatch)

Worst-first ordering (D-17): withheld-wrong -> withheld-open-disagreement -> published-stale (then oldest lastVerified first).
Staleness (D-16): classifyStaleness(fact) -> regulatory 12mo / current 24mo / historical none; lastVerified = max(confirms passes' checkedOn).

Derived states available from deriveVerificationState (02-02): published-confirmed, published-contested, published-stale, withheld-unverified, withheld-in-review, withheld-open-disagreement, withheld-wrong. withheld-in-review and withheld-unverified are the VRFY-03 workflow split (some verification activity but below the bar, versus zero passes) and MUST be counted as separate rows, never merged.
Cache: read .cache/citation-verdicts.json (absent = {}), same Map the gate uses.
Output path: docs/DATA-AUDIT-{YYYY-MM-DD}.md (a dated artefact; generated on demand).
Non-zero-fact assertion required (a zero-fact audit is a false green).
