<objective>
Build the re-verification audit command (`scripts/audit-verification.mjs`): a read-only report generator that walks the corpus, derives each fact's publication state and staleness class via `lib/verification.mjs`, reads the committed existence-verdict cache, and emits a dated, worst-first audit record modelled exactly on DEBT's `docs/DATA-AUDIT.md`. The document is the human triage surface: a counts-by-status table that reports the VRFY-03 workflow states distinctly (including withheld-in-review separately from withheld-unverified), then discrepancy sections ordered wrong -> disagreement -> stale-oldest-first, each carrying the fact's recorded structured measure and pass verdicts, plus a separate reviewer-disagreements section. It also surfaces the two things a fact-state-only table would hide: citations that have rotted (a cached non-RESOLVES verdict, R-18) and citations whose existence check is itself stale past the shared CITATION_TTL_DAYS (R-07), so re-verification of BOTH facts and citations becomes a generated queue rather than a good intention. It warns on future-dated passes (R-25b), lists OFF-derived facts and any that have lost their ODbL link (R-09), and lists authoritative-classed facts for periodic human spot-check (R-30). The audit NEVER writes a verdict or a value: the human adjudicates by editing the fact inline (D-04/D-17).

Purpose: "Verified and re-verified until we are sure." Staleness - of both the fact and its citation - becomes a generated queue, not a good intention.
Output: One standalone read-only script and its test. This plan owns scripts/audit-verification.mjs exclusively.
</objective>

Audit document shape (from DEBT DATA-AUDIT.md, verified template, extended):
  # Food Transparency Data Verification Audit
  **Date:** <ISO or long-form date>
  ## Counts by status         (one row per derived state)
  | Status | Count |
  ## Discrepancies to approve  (ordered worst-first)
  ### Wrong (n)  / ### Open disagreement (n) / ### Stale (n)
  per entry: fact path, current value, the recorded structured measure, the recorded pass verdicts, last-verified date
  ## Citations no longer resolving   (facts whose cited source has a non-RESOLVES cached verdict - rotted citations, R-18)
  ## Citations due for re-check      (cached checkedAt older than CITATION_TTL_DAYS - the citation-staleness queue, R-07)
  ## Data warnings                   (any pass checkedOn or lastVerified in the future, R-25b)
  ## OFF-derived facts (ODbL)        (facts citing a share-alike source; flag any that have lost the ODbL link, R-09)
  ## Authoritative classification spot-check (authoritative-classed facts for periodic human review; a listing only, R-30)
  ## Reviewer disagreements - flag for extra human scrutiny   (facts with a disputes verdict or a measure mismatch)

Worst-first ordering (D-17): withheld-wrong -> withheld-open-disagreement -> published-stale (then oldest lastVerified first).
Staleness (D-16): classifyStaleness(fact) -> regulatory 12mo / current 24mo / historical none; lastVerified = max(confirms passes' checkedOn).

Derived states available from deriveVerificationState (02-02): published-confirmed, published-contested, published-stale, withheld-unverified, withheld-in-review, withheld-open-disagreement, withheld-wrong. withheld-in-review and withheld-unverified are the VRFY-03 workflow split (some verification activity but below the bar, versus zero passes) and MUST be counted as separate rows, never merged.

Cache: read .cache/citation-verdicts.json (absent = {}), same Map the gate uses. Each entry is { verdict, resolvedVia, checkedAt (ISO-8601 date-time), statusCode, snapshotUrl } (the SEAM shape written by 02-05). R-18 reads verdict (non-RESOLVES = rotted); R-07 reads checkedAt (older than the TTL = due for recheck).
CITATION_TTL_DAYS: import the shared constant from lib/verification.mjs (do NOT redefine it - the gate enforces it, the audit surfaces it; same constant, same semantics, value 180).
OFF-derived: use listOffDerived(facts, sources) from lib/referential.mjs (returns the instance paths of facts citing a share-alike source).

Output path: docs/DATA-AUDIT-{YYYY-MM-DD}.md (a dated artefact; generated on demand).
Non-zero-fact assertion required (a zero-fact audit is a false green).
