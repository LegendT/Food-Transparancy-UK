Pagination front-matter (03a-PATTERNS.md "src/product.njk", 03a-RESEARCH.md Pattern 1): the src/_data/products/* cascade is an OBJECT keyed by filename, so resolve: values is MANDATORY (default pagination iterates keys -> strings). Front matter:
  pagination: { data: products, size: 1, alias: product, resolve: values }
  permalink: "/products/{{ product.slug }}/"
  layout: layouts/base.njk
Hyphenated keys (products.spike-01) are invalid Nunjucks dot access; paginate-over-values sidesteps this.

Macro import and call signature (03a-PATTERNS.md): {% from "components/macros.njk" import sourcedValue, sourceNote, caveatBox %}. ALWAYS call with sources.sources (the registry ARRAY inside the wrapper), NEVER sources; findBy returns undefined for a non-array and silently breaks TRUST-03. Thread entityType "product" for product facts and "timeline" for change events (staleness class; timeline events are historical, never-stale).

Product fact fields (schemas/product.schema.json, spike-02.json): manufacturer (SourcedValue), ingredientsText (SourcedValue, the declared-recipe fact of record), plus the D-15 `ingredients` metadata array (navigation only, NOT the declared recipe). Nutrition and allergens are OWNED BY PLAN 04 - do not render them here.

Recipe-history (03a-RESEARCH.md "Code Examples", 03a-UI-SPEC.md "Recipe History Absent State"): set events = timelineByProduct[product.id] (global from 03a-01). For each event.changes[] render documentedChange and statedReason via sourcedValue with entityType "timeline"; labelledInference is NOT a fact (no sources/claimType) - render as labelled illustration only, never through the trust component. Empty state: a calm #f3f2f1 panel with the verbatim UI-SPEC copy ("No recipe changes recorded yet" + body), never omitted, never a withheld/error card. The conditional "full timeline" link renders only where a Phase 8 timeline page exists (none yet -> omit).

Heading outline (UI-SPEC G4): exactly one h1 (product name); h2 sections in D-05 order Ingredients, Nutrition, Allergens, Manufacturer, Sources, Recipe history. This plan authors Ingredients, Manufacturer, Sources, Recipe history and LEAVES the Nutrition and Allergens h2 sections for plan 04 to insert in order. Any sub-grouping uses h3; never skip a level.

Sources roll-up (PROD-04, D-07): a page-level list of the distinct source records cited by the page's facts, resolved by id against sources.sources (reuse sourceNote / the findBy name-url idiom); the per-fact disclosure remains.

Output encoding: autoescaping ON; never `| safe` on a value or source name; rel="noopener" on external links. No raw `.value` anywhere in this template (check-render-safety) - values render only through the macro.
